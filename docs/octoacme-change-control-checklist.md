# OctoAcme — Change Control Checklist

## Purpose
Provide a standardized process for requesting, reviewing, and approving changes to project scope, requirements, or schedule.

## When to Use
Use this checklist whenever a change is proposed that affects:
- Project scope or deliverables
- Timeline or milestones
- Budget or resource allocation
- Requirements or acceptance criteria

## Change Control Process Checklist

### Submission Phase
- [ ] Change request form completed with all required fields
- [ ] Business justification provided
- [ ] Initial priority assigned
- [ ] Request submitted to Change Control Lead

### Review Phase
- [ ] Change request logged in change log
- [ ] Impact assessment initiated with affected teams
- [ ] Technical feasibility reviewed
- [ ] Schedule impact analyzed
- [ ] Budget impact calculated (if applicable)

### Approval Phase
- [ ] Review meeting scheduled (if needed)
- [ ] Stakeholder input gathered
- [ ] Decision documented with rationale
- [ ] Approval escalated to Project Sponsor (if threshold exceeded)

### Implementation Phase
- [ ] Approved change communicated to all affected parties
- [ ] Project plan updated to reflect change
- [ ] Backlog updated with new/modified items
- [ ] Risk register updated (if applicable)
- [ ] Change log updated with final status

### Closure Phase
- [ ] Change implementation verified
- [ ] Documentation updated
- [ ] Change request closed in change log

---

## Approval Thresholds

| Impact Level | Approver |
|--------------|----------|
| Minor (< 5% schedule/budget impact) | Change Control Lead |
| Moderate (5-15% schedule/budget impact) | Project Manager + Change Control Lead |
| Major (> 15% schedule/budget impact) | Project Sponsor |
